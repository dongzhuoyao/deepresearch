## Workspace Areas

- `topic.txt`: research topic
- `config.yaml`: effective workspace config
- `idea/`: idea generation and debate artifacts
- `plan/`: experiment plans and task definitions
- `exp/`: experiment code, logs, and results
- `writing/`: sections, critique, figures, LaTeX outputs
- `reflection/`: lessons learned and evolution inputs
- `logs/`: event logs and status artifacts

## Practical Checks

- Before changing anything, read workspace status.
- Before advancing anything, read the next action.
- After completing anything material, record the result.
- If a workspace looks inconsistent, trust the orchestrator output over memory.
